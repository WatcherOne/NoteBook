jar包 就是 引入的依赖【通常读作jar包】

Maven
  1. 基于Java平台的项目构建、依赖管理和项目信息管理
  2. 统一开发规范与工具，统一管理jar包
  1. 帮助快速发布项目
Spring Boot
  1. 起步依赖: 将具备某种功能的坐标打包到一起，解决了Spring的jar包冲突问题
  2. 自动配置:在底层给我们自动做了一些配置，所以SpringBoot不编写配置为文件也可以正常运行
  3. 一个服务于框架的框架，服务范围是简化配置文件
  4. 可以以jar包形式独立运行，java -jar XX.jar
  5. 创建独立的Spring项目；内置Tomcat和Jetty容器
Mybatis
  1. 一个基于Java的持久层框架，管理数据库操作的框架
  2. sql写在xml里面，方便统一管理和优化；解除sql与程序代码的耦合；提供映射标签；支持编写动态sql
  3. 管理事务的两种形式：使用JDBC；使用MANAGED，让程序实现管理
  1. Spring Boot 是对 Spring的简化配置与扩展功能（自动配置的优化）
  2. Spring Boot 也可以直接通过创建Maven来生成的（不用选骨架）
  3. Spring Boot  @SpringBootApplication 注解也表明了 注册其它注解为上下文组件，会扫描当前所在包所有文件
  4. resources里面有个 application.properties / application.yml (配置文件类型不同而已，多种写法，功能一样)
    properties 文件 优先级 高于 yml 文件
    JVM环境变量  >  操作系统环境变量  >  application.properties  >  application.yml
  5. jar 包 spring-boot-starter-web 加入 api 功能，本身它自己也依赖了 核心 spring-boot-starter 故直接用这个即可
maven项目引用 springBoot 来形成 spring 项目
<parent>
    <groupId>org.springframework.boot</groupId>
    <artifactId>spring-boot-starter-parent</artifactId>
    <version>2.4.2</version>
</parent>

注解说明
@bean
  1. 加上注解，定义或重写bean方法
  2. 定义bean也可以 @importResource(“bean.xml”) 在resources文件夹下定义xml文件
@Configuration
告诉 SpringBoot 这是一个配置类，可以让SpringBoot扫描到
等同于Spring的XML配置文件
