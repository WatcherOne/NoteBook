  2. 1996年JDK1.0正式发布，2014年JavaSE8.0发布，现在大多数仍然以8为主（jdk1.8 就是 jdk8）
           2023年- Java20（平均每半年更新一次，9月,3月）
      3. 跨平台是Java语言的核心优势
      4. Java各版本
JavaSE: 标准版，用于开发一般个人计算机上的应用程序，提供了丰富的API
JavaEE: 企业版，定位在服务器端的应用，增加了用于服务器开发的类库
JavaME：微型版，定位在消费性电子产品的应用上，不常用
Java的源文件，先利用编译器（javac）将源文件编译成字节码文件（.class）
最后利用虚拟机（解释器JVM）解释执行
JVM（Java Virtual Machine）：用于执行bytecode字节码的虚拟计算机
           不同操作系统有不同版本JVM，屏蔽了底层运行平台的差别，是实现跨平台的核心
JRE（Java Runtime Environment）：Java运行时环境，包含Java虚拟机，库函数等
JDK（Java Development Kit）：Java开发工具包，包含JRE，编译器和调试器等
在idea中有 Project SDK：表示软件开发工具包，选择JDK

  1. java文件名必须与 内部calss名一致
  2. javac xxx.java 编译时 需要将文件后缀名带上
  3. java xxx 运行时不需要 自动去找.calss的字节码文件
  4. javac 编译时，有中文时，编码规则不同可能编译报错
  5. java区分大小写
  6. java是面对对象的，所有代码必须放到类里面
  7. 源文件可以包含多个类，编译器为每个类都会生成一个字节码文件
  8. 注释不会出现在字节码文件中
