  1. Maven Projects 说明
1.Lifecycle【生命周期】：展开后里面是一些可执行的方法(包括 compile 编译,clean 清除,install 打包发布……)
3.Dependencies【依赖】：引入的依赖

  1. pom.xml文件说明
A. 可以被继承，内部设置 <packaging>pom</packaging>, 项目里面只有pom.xml 作为父工程
   其他 module 项目配置 parent 为 父工程来继承！
B. 子孙继承；在 pom.xml 中设置 packaging 标明是作为父工程 里面的 src test 要去掉（同上，但感觉这样没有意义了）
C. <dependencyManagement></dependencyManagement> 将依赖包起来，设置依赖管理，解决不是所有子工程都继承父工程的所有依赖
   此时，父工程把依赖管理起来后，子工程要单独引一下依赖只是不需要设置版本号，可以替换继承来的版本号
D. 配置<properties></properties> 相当于设置变量可以 整体管理 版本号等参数
